# Phase 3 — P1 #73 clock fix + #44 devtools panel (implementer)

Cleared the reviewer's P1 (my own #23 regression), then built #44. Both In Progress.

## [review][P1] #73 — clock-pin broke the live eval (commit `303a513`)

My #23 bonus fix (global `vitest.setup.ts` freezing Date for snapshot determinism)
silently broke `npm run eval`: the runner classifies no_build by `Date.now() <
deadline`, and a frozen Date never expires → every refusal case hung until the
outer timeout (reviewer saw rj-02 @ 9,133,051ms). Fix: `vi.useRealTimers()` at the
top of the live suite (scoped to that file; render.test keeps the pin) + a CANARY
— a 15ms real delay must advance Date.now(), else fail in ms with a pointer here.
Verified `EVAL_IDS=p0-01,rj-02` → rj-02 no_build, whole run 104s (was 2.5h).
**Lesson: a global test fixture is cross-cutting — grep Date.now() consumers
before freezing a clock.**

## #44 — devtools panel (commit `2eb058e`)

New package **`@workspace-engine/devtools`** (peer react + @tanstack/react-query).
Architecture: a framework-free **event bus** — `recordSpec` / `recordVerdict` /
`recordQuery` push to a ring buffer, `useDevtoolsLog` reads it — so the SDK never
imports the panel (that's how it tree-shakes out of prod).

- `WorkspaceDevtools`: floating, dev-only overlay, three tabs:
  - **Spec** — current WorkspaceSpec (pretty JSON) + history of every spec.
  - **Verdicts** — each validator verdict + per-field reasons (build/clarify/reject).
  - **Queries** — binding → fetch → rows timeline with client-observed timing.
- `DevtoolsQueryReporter` (mounted inside WorkspaceProvider) subscribes to the
  react-query cache, times each fetch (fetching→settled), and reports to the bus.
- Demo wiring: GeneratedWorkspace records the gate verdict + built spec (deduped,
  skipping mid-stream shape rejects); /create mounts the panel behind a NODE_ENV
  guard. 3 component tests.

**Verified LIVE (dev, headed):** flagship prompt →
- Spec: the full GroupedBoard spec JSON.
- Verdicts: an intermediate `REJECT "blk_board needs a 'groups' query but derives
  'rows' → add a groupBy"` (a streaming-intermediate spec) then `BUILD` — the exact
  "contract vs model problem" inspectability the card is about.
- Queries: `success blk_board · case · 5 rows · 1ms · filter risk,dueDate · groupBy
  analyst`.

**Prod exclusion verified**: `next build` output has none of the devtools strings
(NODE_ENV guard + the component's own prod null-return + tree-shaking). Monorepo
build (4 pkgs) + check-types (6) + lint green; demo 55/1skip.

### Gotchas
- `Omit<Union, K>` collapses to common keys — use a distributive `T extends T ?
  Omit<T,K> : never` so each event member keeps its fields.
- The package has no `@types/node`; the prod check uses `declare const process`.
- A new workspace package needs `npm install` at root (link + deps) AND in the
  demo (link the file: dep) before it resolves.
